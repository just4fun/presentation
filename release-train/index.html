# Release Train #### Zoro Jiang #### Feb 20 2023
# What

Software Release Life Cycle

An initiative to align and manage the software releases that happen in company to EMT project.

# Why
- Lack of Tracking - Lack of Transparency - Human Error - Production Incidents
# When

EMT_2022_01.00 was started at June 3 2022.

We had 14 releases in 2022.

EMT_2023_02.00 will be released at early Mar.

# How

- Track the production release via JIRA Fix Version

https://jira.aligntech.com/projects/APW/versions/233251

- Track the version of production release package

https://www.atlassian.com/git/tutorials/comparing-workflows/gitflow-workflow

- Regression test for each release train - QA

- Communication with each dev & Release branch preparation & Release - Release Train Engineer

- Document the approvals / Testing Report / Testing Sign-off

# Improvements

- Release schedule is fixed

Marketing teams won't introduce urgent ticket during a sprint, as they know the train already departed.

- PROD support tickets reduced

# Side Effects

- Regression test costs much time

Until we introduce automation to cover.

- One more role with additional responsibility - Release Train Engineer

# FAQ

How the PROD issue release works with release train?

- Created a new fix version on JIRA. e.g. EMT_2023_01.01
- Follow git flow, create hotfix then merge to master. After release, merge it back to develop branch.

What about if QA doesn't have enough capacity to run regerssion test?

- Release train is controlled by dev team. If no enough capacity to run regression test, We will only run functional tests, or we ask BA to help on testing, or even postpone the release train (need to align with marketing team ahead).
